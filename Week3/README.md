# 🌿 GE338-Lab-3

## 🛰️ ภาพรวมของงาน
โครงการนี้เป็นงาน **Lab 3: Machine Learning สำหรับการจำแนกการใช้ที่ดิน** โดยใช้ภาพดาวเทียม Sentinel-2 และ training polygons ของจังหวัดนครนายก เพื่อสร้างแผนที่การใช้ที่ดินและประเมินผลด้วยวิธีเชิงสถิติและเชิงภูมิสารสนเทศอย่างเป็นระบบ งานนี้ออกแบบตามโจทย์ที่กำหนดให้ต้องอธิบาย **training strategy**, การเปรียบเทียบอัลกอริทึมอย่างน้อย 2 วิธี, การวิเคราะห์ **feature importance**, และการประเมิน **uncertainty** ของแผนที่ที่จำแนกได้อย่างชัดเจน

## 🎯 วิธีที่เลือก
งานนี้เลือกใช้ **Random Forest** เป็นโมเดลหลัก และใช้โมเดลเปรียบเทียบอีกอย่างน้อย 1 วิธีใน notebook เปรียบเทียบ เพื่อให้สอดคล้องกับโจทย์ของ Lab ที่ต้องทดลองอย่างน้อย 2 อัลกอริทึม พร้อมรายงานมากกว่า Overall Accuracy เพียงอย่างเดียว โดย workflow หลักประกอบด้วยการอ่านภาพดาวเทียม, อ่าน training polygons, ดึงค่าพิกเซลจากแต่ละ class, สร้างชุด feature, แบ่ง train/validation, ฝึกโมเดล, ประเมิน confusion matrix และสร้างแผนที่ผลลัพธ์

สำหรับพื้นที่ศึกษา เลือก **จังหวัดนครนายก** เพราะมีการใช้ที่ดินหลากหลาย ทั้งป่าไม้ แหล่งน้ำ พื้นที่เกษตร และพื้นที่ชุมชน ทำให้เหมาะกับการทดสอบการจำแนกหลาย class ในภาพเดียวกัน ความหลากหลายนี้ยังช่วยให้เห็นจุดแข็งและจุดอ่อนของแต่ละอัลกอริทึมได้ชัดขึ้น โดยเฉพาะเมื่อ class บางคู่มีลักษณะเชิงสเปกตรัมใกล้เคียงกัน

## 🧪 Training Strategy
การออกแบบ training strategy เป็นหัวใจของ Lab นี้ เนื่องจากโจทย์กำหนดให้นักศึกษาต้องคิดเรื่อง class definition, แหล่งที่มาของ training samples, วิธีแบ่ง train/validation และการเลือก feature ก่อนเริ่มเขียนโค้ดจริง ข้อมูลฝึกสอนในงานนี้มาจากไฟล์ `Trainning.shp` ซึ่งมี field สำคัญ เช่น `Classname`, `Classvalue` และ `Count` จึงสามารถใช้เป็น label สำหรับฝึกโมเดลได้โดยตรง

ในงานนี้ใช้ **7 class** ได้แก่ Bareland, Buildup, Fieldcrop, Forest, Orchard, Perennial และ Water เนื่องจากข้อมูลฝึกสอนเดิมของผู้จัดทำแยก class เหล่านี้ไว้อย่างชัดเจนและสะท้อนบริบทจริงของนครนายก การใช้ 7 class แม้จะมากกว่าช่วงแนะนำ 4–6 class เล็กน้อย แต่ช่วยรักษาความละเอียดเชิงภูมิศาสตร์ของพื้นที่เกษตรและพืชยืนต้นในพื้นที่ศึกษาไว้ได้

การแบ่งข้อมูลใช้แนวทาง **random split 80/20** เป็นค่าเริ่มต้นเพื่อให้ workflow รันได้ง่ายและเปรียบเทียบโมเดลได้ตรงไปตรงมา อย่างไรก็ตาม ในเชิงวิชาการต้องยอมรับว่าแนวทางนี้อาจทำให้ค่าความแม่นยำสูงเกินจริงได้ หากข้อมูลฝึกและข้อมูลทดสอบอยู่ใกล้กันมากในเชิงพื้นที่ ดังนั้นใน README นี้จึงอภิปรายเพิ่มเติมเรื่อง **spatial autocorrelation** และข้อแตกต่างระหว่าง random split กับ spatial cross-validation ไว้ด้วย

## 🌈 Feature ที่ใช้และเหตุผล
ในส่วนของ feature งานนี้ใช้ทั้ง **spectral bands** และ **spectral indices** เพื่อให้ตอบโจทย์ Lab ที่ถามโดยตรงว่า “ถ้าใช้แค่ bands ปกติ เทียบกับ bands + indices จะต่างกันแค่ไหน” โดย bands ที่เน้นใช้คือชุดหลักของ Sentinel-2 เช่น Blue, Green, Red, NIR รวมถึง bands เพิ่มเติมที่ช่วยแยกดิน พืช และความชื้นได้ดีขึ้น ส่วน indices ที่คำนวณเพิ่ม เช่น NDVI, GNDVI, NDWI และ SAVI ถูกใช้เพื่อดึงความหมายเชิงกายภาพของพืชพรรณและน้ำออกมาอย่างชัดเจนขึ้น

เหตุผลสำคัญของการเพิ่ม indices คือ class หลายคู่ในงานนี้ เช่น Fieldcrop, Orchard และ Perennial มีความใกล้เคียงกันทางสเปกตรัม หากใช้ bands ดิบเพียงอย่างเดียว โมเดลอาจสับสนได้ง่าย การเพิ่มดัชนีจึงเป็นการใส่ความรู้ทางรีโมตเซนซิงเข้าไปในโมเดล เพื่อช่วยให้การแยกกลุ่มพืชพรรณและแหล่งน้ำมีประสิทธิภาพขึ้น

## 🤖 ผลการเปรียบเทียบอัลกอริทึม
ผลจากรายงานเดิมของผู้จัดทำชี้ว่า **Random Forest** ให้ผลดีที่สุดในพื้นที่นครนายก โดยมีค่า Overall Accuracy และ Kappa สูงกว่า Maximum Likelihood และ ISODATA อย่างชัดเจน ข้อค้นพบนี้สอดคล้องกับธรรมชาติของข้อมูล land use/land cover ที่มักมีความไม่เป็นเชิงเส้นและมี class overlap สูง โดยเฉพาะในพื้นที่เกษตรและพื้นที่ผสม การใช้โมเดลแบบ ensemble tree จึงยืดหยุ่นกว่าวิธีเชิงสถิติแบบดั้งเดิม

ใน notebook หลัก (`lab3_classification.ipynb`) มี workflow สำหรับฝึกโมเดลและสร้างผลลัพธ์ทั้งหมด ส่วน notebook เปรียบเทียบ (`lab3_comparison.ipynb`) ใช้สำหรับจัดตารางเปรียบเทียบผล, อ่านไฟล์ summary ที่ถูกบันทึกจาก notebook หลัก, และเขียนข้อสรุปเชิงตีความให้พร้อมนำไปใส่ในรายงานหรือ README

การประเมินผลไม่ได้ดูเพียง Overall Accuracy แต่รวมถึง **Producer’s Accuracy, User’s Accuracy, Kappa และ F1-score ต่อ class** ด้วย เพื่อให้เห็นว่า class ใดถูกจำแนกได้ดีจริง และ class ใดที่ยังมีปัญหา แม้ค่า OA โดยรวมจะดูสูงก็ตาม

## 🌲 Feature Importance
ในส่วนของ Random Forest งานนี้ให้ความสำคัญกับการดึง **feature importance** ออกมาวิเคราะห์ ไม่ใช่เพียงรายงานว่าคุณลักษณะใดเด่นที่สุด แต่ต้องตีความต่อด้วยว่าความสำคัญนั้นสอดคล้องกับความเป็นจริงทางกายภาพหรือไม่ เช่น feature ที่เกี่ยวข้องกับ NIR หรือ vegetation indices มักมีบทบาทมากในการแยกป่าไม้และพืชพรรณ ขณะที่ feature ด้านความชื้นหรือ SWIR มักช่วยแยกน้ำหรือดินเปลือยได้ดี

นอกจากนี้ workflow ยังรองรับการทำ **ablation test** โดยตัด feature ที่สำคัญต่ำออกแล้วฝึกใหม่ เพื่อดูว่าความแม่นยำเปลี่ยนแปลงอย่างไร หากผลลดลงเพียงเล็กน้อย แสดงว่าบาง feature อาจซ้ำซ้อนและสามารถลดจำนวนลงเพื่อให้โมเดลเรียบง่ายขึ้นโดยไม่เสียประสิทธิภาพมาก

## 🌫️ ความไม่แน่นอนและข้อจำกัดที่พบ
ข้อจำกัดสำคัญของงานนี้อยู่ที่ class บางคู่มีลักษณะใกล้เคียงกันมาก โดยเฉพาะกลุ่มเกษตรและพืชยืนต้น เช่น Fieldcrop, Orchard และ Perennial ทำให้เกิดความสับสนใน confusion matrix สูงกว่ากลุ่ม Forest หรือ Water ที่มักมีลักษณะเด่นชัดกว่า นอกจากนี้ training data ยังอาจมีความไม่สมดุลระหว่าง class ซึ่งมีผลต่อการเรียนรู้ของโมเดลโดยตรง

อีกประเด็นที่สำคัญคือ **spatial autocorrelation** เพราะถ้า train/test อยู่ใกล้กันมาก ค่าความแม่นยำที่ได้อาจสูงกว่าความจริงเมื่อนำโมเดลไปใช้กับพื้นที่ใหม่ งานนี้จึงอธิบายไว้อย่างชัดเจนว่า random split เป็นเพียงจุดเริ่มต้นของ workflow แต่ไม่ใช่วิธีประเมินที่เข้มงวดที่สุด

ในเชิง uncertainty โมเดลจะไม่แน่ใจมากในบริเวณที่เป็นพื้นที่เกษตรผสม ขอบแปลง หรือพื้นที่รอยต่อระหว่าง class ซึ่งสามารถวิเคราะห์ต่อได้ด้วย probability map จาก Random Forest เพื่อระบุบริเวณที่ class probability ต่ำหรืออันดับ 1 กับอันดับ 2 มีค่าต่างกันน้อยมาก

## 🔍 สิ่งที่ค้นพบ
จากการออกแบบและ workflow ของงานนี้ มีข้อค้นพบสำคัญคือ **Random Forest เป็นตัวเลือกที่เหมาะสมที่สุด** สำหรับบริบทของจังหวัดนครนายกเมื่อ training data มีจำกัดและโครงสร้างข้อมูลซับซ้อน การเพิ่ม spectral indices มีแนวโน้มช่วยให้การแยกกลุ่มพืชพรรณและน้ำดีขึ้น และการอ่านผลจาก confusion matrix แบบราย class มีประโยชน์มากกว่าการดู OA เพียงอย่างเดียว เพราะช่วยชี้ว่าปัญหาหลักจริง ๆ อยู่ที่ class ใด

งานนี้ยังแสดงให้เห็นว่าการทำแผนที่ที่ “ดูดี” ไม่ได้แปลว่าเชื่อถือได้ทุกจุด การรายงานข้อจำกัดอย่างซื่อสัตย์ โดยเฉพาะในพื้นที่ที่โมเดลไม่แน่ใจ เป็นส่วนสำคัญของงานด้าน Geographic Data Science และรีโมตเซนซิงในโลกจริง
